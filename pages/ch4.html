Task 1: dff

In this exercise you are required to implement a D Flip Flop with the following input, output and functionalities.

Inputs:

"d": a 1-bit wide input for data. "clk": a 1-bit wide input for clock. "rst_n": a 1-bit wide input for reset (active low).

Output:

The output of the D flip flop is a single bit and is named "q".

Functionality:

A D flip-flop is a type of sequential logic circuit that stores one bit of data. It has a data input (denoted as "d"), a clock input (denoted as "clk"), a reset input (denoted as "rst_n"), and an output (denoted as "q").

The D flip-flop operates in the following way:

  • When the clock input "clk" rises from 0 to 1, the D flip-flop latches the data input "d" onto its output "q".

  • The output "q" holds this value until the next clock edge arrives, at which point the D flip-flop latches in the new value of "d".

  • If the reset input "rst_n" is 0, the output "q" is forced to 0, regardless of the value of the data input "d". When "rst_n" goes back to 1, the D flip-flop resumes normal operation.

Testbench:

  1. Instantiate the D flip-flop module: In your testbench, you should instantiate the D flip-flop module you want to test. Connect the inputs and outputs of the module to signals in the testbench.

  2. Generate a clock signal: Create a clock signal using an "always" block that changes the value of the "clk" signal on each rising edge of the clock. The period of the clock should be long enough to allow the D flip-flop to respond, but not so long that your simulation takes too much time to run.

  3. Apply inputs: In the initial block of your testbench, apply initial values to the inputs of the D flip-flop. This will ensure that the D flip-flop starts in a known state. You can also use this block to apply a reset pulse to the "rst_n" input to ensure that the output "q" is initially set to 0.

  4. Check the output: Use "assert" statements to check that the output "q" of the D flip-flop behaves as expected. For example, you can check that the output follows the value of the input "d" on each clock edge, and that the output is set to 0 when the reset input "rst_n" is low.

  5. Stimulate the inputs: Use "repeat" or "forever" loops to apply different test patterns to the input "d" of the D flip-flop. This will allow you to test different scenarios and ensure that the D flip-flop works correctly under different conditions.

  6. Verify the results: After running the simulation, check the simulation log for any errors or warnings. You can also use waveform viewers to visualize the signals and ensure that they behave as expected.


Notes:

  • always_ff:
    "always_ff" is a SystemVerilog construct used for sequential logic.
always_ff @(posedge clk or negedge rst_n)
This code is executed on every positive edge of the "clk" input and on every negative edge of "rst_n" input.
